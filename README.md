این پروژه بیشتر روی پیاده‌سازی یک سرویس سبد خرید با معماری Clean Architecture و CQRS تمرکز داشت. سعی کردم ساختار پروژه تمیز و قابل توسعه باشد و منطق هر بخش در لایه خودش قرار بگیرد. برای مدیریت Command و Query از MediatR استفاده شد و Eventهای داخلی پروژه هم با MediatR مدیریت شدند. RabbitMQ هم در پروژه راه‌اندازی و کانفیگ شد، اما در این تسک عملاً از آن برای انتشار Eventها استفاده نشد.

بزرگ‌ترین چالش پروژه برای من خود کدنویسی نبود، بلکه درآوردن معماری درست از روی توضیحات تسک بود. فایل Wordی که به عنوان مستند پروژه داده شده بود، خیلی نامرتب و بعضی قسمت‌هایش گمراه‌کننده بود و مشخص نبود دقیقاً انتظار چه ساختاری را دارند. به همین خاطر بخش زیادی از زمان صرف تحلیل نیازمندی‌ها و رسیدن به یک معماری درست شد تا صرفاً نوشتن کد.


طراحی ساختار پروژه بر اساس Clean Architecture و تفکیک لایه‌های Domain، Application، Infrastructure و API.
پیاده‌سازی الگوی CQRS و جداسازی کامل Command و Queryها.
طراحی موجودیت Basket و پیاده‌سازی منطق دامنه مثل اضافه کردن کالا، حذف کالا، تغییر تعداد و خالی کردن سبد خرید.
پیاده‌سازی Repositoryها و ارتباط با دیتابیس از طریق Entity Framework Core.
طراحی ServiceResult برای برگرداندن پاسخ‌های یکپارچه از سرویس‌ها و مدیریت خطاها.
استفاده از MediatR برای مدیریت Command، Query و همچنین Eventهای داخلی پروژه.
راه‌اندازی و کانفیگ RabbitMQ و MassTransit برای آماده بودن زیرساخت پیام‌رسانی، هرچند در این تسک Eventها با MediatR مدیریت شدند.
راه‌اندازی Redis و اتصال پروژه به آن برای استفاده به عنوان Cache.
داکرایز کردن پروژه و ساخت Dockerfile و docker-compose برای اجرای API، SQL Server، Redis و RabbitMQ به صورت یکپارچه.
رفع مشکلات مربوط به اتصال سرویس‌ها داخل Docker و تنظیم Connection Stringها.
مدیریت Dependency Injection و ثبت سرویس‌ها به شکلی که وابستگی بین لایه‌ها حفظ شود.
پیاده‌سازی APIهای مربوط به سبد خرید و استانداردسازی خروجی آن‌ها.
