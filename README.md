# توضیحات پروژه

این پروژه روی پیاده‌سازی یک سرویس **سبد خرید (Basket Service)** با استفاده از **Clean Architecture** و **CQRS** تمرکز دارد. هدف اصلی، طراحی یک ساختار تمیز، قابل توسعه و قابل نگهداری بود تا هر بخش از منطق پروژه در لایه‌ی مناسب خودش قرار بگیرد.

برای مدیریت Command و Query از **MediatR** استفاده شده و Eventهای داخلی پروژه نیز با همین ابزار مدیریت می‌شوند. همچنین **RabbitMQ** و **MassTransit** در پروژه راه‌اندازی و کانفیگ شده‌اند، اما در این تسک از آن‌ها برای انتشار Eventها استفاده نشده است.

## چالش‌های پروژه

بزرگ‌ترین چالش این پروژه خود کدنویسی نبود، بلکه تحلیل نیازمندی‌ها و رسیدن به یک معماری درست بود. مستند پروژه که به صورت فایل Word ارائه شده بود، ساختار منظم و شفافی نداشت و بعضی قسمت‌های آن برداشت‌های متفاوتی ایجاد می‌کرد. به همین دلیل بخش قابل توجهی از زمان صرف تحلیل نیازمندی‌ها، طراحی معماری و تصمیم‌گیری درباره ساختار پروژه شد.

## کارهای انجام شده

- طراحی ساختار پروژه بر اساس **Clean Architecture**
- تفکیک لایه‌های Domain، Application، Infrastructure و API
- پیاده‌سازی الگوی **CQRS** و جداسازی کامل Command و Query
- طراحی موجودیت Basket و پیاده‌سازی منطق دامنه
  - افزودن کالا به سبد خرید
  - حذف کالا
  - تغییر تعداد کالا
  - خالی کردن سبد خرید
- پیاده‌سازی Repositoryها با استفاده از Entity Framework Core
- طراحی `ServiceResult` برای استانداردسازی خروجی سرویس‌ها و مدیریت خطاها
- استفاده از MediatR برای مدیریت Command، Query و Eventهای داخلی
- راه‌اندازی و کانفیگ RabbitMQ و MassTransit
- راه‌اندازی Redis و اتصال آن به پروژه برای استفاده به عنوان Cache
- Dockerize کردن پروژه
- ایجاد `Dockerfile` و `docker-compose`
- راه‌اندازی SQL Server، Redis، RabbitMQ و API داخل Docker
- رفع مشکلات ارتباط بین سرویس‌ها و تنظیم Connection Stringها
- مدیریت Dependency Injection و ثبت سرویس‌ها
- پیاده‌سازی APIهای مربوط به سبد خرید
- استانداردسازی خروجی APIها
